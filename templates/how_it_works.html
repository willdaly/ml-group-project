How It Works

An overview of the dataset, features, models, and preprocessing behind the detection engine.

Dataset: NSL-KDD

The project uses the NSL-KDD dataset, a refined benchmark derived from KDD Cup 1999 network intrusion data. Each row describes a single network connection with protocol details, byte counts, error rates, host-level statistics, and a label.

The dataset includes two files: KDDTrain+.txt (training) and KDDTest+.txt (evaluation), stored locally under data/nsl-kdd/ so the project runs fully offline.

Input Features

Each connection record contains fields such as:

  • Protocol type — tcp, udp, icmp
  • Service — http, ftp, smtp, private, etc.
  • Flag — connection status (SF, S0, REJ, etc.)
  • Byte counts — src_bytes, dst_bytes
  • Error rates — serror_rate, srv_serror_rate, dst_host_serror_rate
  • Connection counts — count, srv_count, dst_host_count
  • Login indicators — num_failed_logins, logged_in, root_shell

Categorical fields are one-hot encoded. Numeric fields are scaled and imputed.

Binary Classification

The first task separates traffic into two classes: normal and attack.

Three models are trained and evaluated:

  • Logistic Regression
  • Random Forest
  • Gradient Boosting

The best model is selected by F1 score on the test set.

Multiclass Attack Categories

Once traffic is flagged as an attack, a second Random Forest model classifies it into one of five NSL-KDD categories:

  • DoS — Denial of Service
  • Probe — Surveillance / scanning
  • R2L — Remote to Local unauthorized access
  • U2R — User to Root privilege escalation
  • Normal — Benign traffic

Preprocessing

The pipeline applies:

  • One-hot encoding for categorical features (protocol_type, service, flag)
  • Median imputation for missing numeric values
  • Standard scaling for numeric features
  • Safe defaults for missing columns at inference time

This ensures the demo can score compact JSON inputs without requiring all 41 original NSL-KDD columns.

Inference & Explanation

Each prediction returns:

  • Binary class — normal or attack
  • Attack category — when classified as an attack
  • Confidence score — from predict_proba when available
  • Analyst explanation — a plain-English summary of which features contributed to the classification
