1

NSL-KDD Cybersecurity Detection Engine

A class ML pipeline refactored into a reusable intrusion-detection engine with saved models, inference services, incident reports, and a browser demo.

2

The Problem

Network defenders need fast ways to separate normal traffic from suspicious activity. Manual inspection doesn't scale when thousands of connections arrive every second.

3

Dataset: NSL-KDD

  • Tabular network connection records
  • Protocol, service, byte counts, error rates, host-level statistics
  • Labels: normal traffic and multiple attack types
  • Categories: DoS, Probe, R2L, U2R, Normal
4

Modeling Approach

Binary task: normal vs attack — Logistic Regression, Random Forest, Gradient Boosting. Best selected by F1.

Multiclass task: attack category — Random Forest over five categories.

Preprocessing: one-hot encoding for categoricals, median imputation and scaling for numerics.

Live Demo

  2. Open /demo
  3. Load benign example → score → see "Normal Traffic"
  4. Load attack example → score → see "Attack Detected" with category and explanation
  5. Call /report/incidents to summarize predictions

8

Limitations

  • NSL-KDD is a benchmark, not modern production traffic
  • Explanations are concise analyst summaries, not full causal attributions
  • Training endpoint is synchronous (demo use only)
  • Models are classical scikit-learn baselines for class-project clarity
9

Future Work

  • Richer explanations (permutation importance, SHAP)
  • Uploaded CSV batch scoring
  • Asynchronous training jobs and run history
  • Model cards with dataset provenance and caveats
  • Autonomous agent wrapper around the service layer
